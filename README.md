# SQL-Sales-Analysis-Using-Relational-Joins

## Project Overview
This project demonstrates the use of SQL INNER JOIN and LEFT JOIN to analyze sales data from a relational database.
The goal is to combine multiple tables and extract meaningful business insights related to customers, products, categories, and revenue.

## Database Used
A sample E-commerce relational database containing:
1. Customers
2. Orders
3. Order Items
4. Products
5. Categories

## Tools & Technologies
1. MySQL
2. MySQL Workbench

## Key Analysis Performed
1. Customer and order analysis using INNER JOIN
2. Identification of customers with no purchases using LEFT JOIN
3. Product-wise and category-wise revenue calculation
4. Region-based and date-based sales analysis
5. Top-performing products identification
